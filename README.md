🛒 E-commerce E2E Automation (Ohou & Musinsa)**오늘의집(Ohou)**과 **무신사(Musinsa)**의 핵심 구매 프로세스(검색 → 필터 → 장바구니)를 검증하는 E2E 테스트 자동화 프로젝트입니다.로컬 환경뿐만 아니라 GitHub Actions를 통한 CI 환경에서도 안정적으로 동작하는 파이프라인을 구축하는 데 중점을 두었습니다.📸 Test Results (CI/CD Pipeline)"From Failure to Success" > 초기에는 봇 탐지 및 환경 이슈로 인해 실패(Fail)했으나, 원인 분석 및 설정을 통해 최종적으로 안정적인 성공(Success) 파이프라인을 구축했습니다.(위 스크린샷은 실제 GitHub Actions 실행 로그입니다.)🔥 Key Troubleshooting & Features이 프로젝트를 진행하며 마주친 4가지 핵심 기술적 이슈와 해결 과정입니다.1. 🔐 보안 (Security): 민감 정보 관리문제: 자동 로그인을 위한 쿠키 파일(auth.json)에는 민감한 세션 정보가 포함되어 있어, Public Repository에 업로드할 수 없음.해결:OpenSSL을 이용해 쿠키 파일을 암호화(auth.cipher)하여 커밋.복호화 키는 GitHub Secrets에 등록하고, CI 실행 시점에 복호화하여 사용하도록 구성.2. 🤖 봇 탐지 우회 (Anti-Bot / WAF)문제: 로컬에서는 정상 작동하던 '오늘의집' 테스트가 CI 서버(Headless Chrome)에서는 **"Access Denied"**로 차단됨.해결: playwright.config.ts에 강력한 스텔스(Stealth) 설정 적용.User-Agent를 일반 브라우저와 동일하게 설정.Referer, Accept-Language 등 리얼한 헤더 추가.브라우저 구동 Args에 --disable-blink-features=AutomationControlled 옵션 추가.3. 🌍 접속 환경 및 리다이렉트 (Geo-blocking)문제: GitHub Actions 서버(해외 IP)에서 '무신사' 접속 시, 강제로 **글로벌 사이트(Global)**로 리다이렉트되어 한국 전용 테스트 시나리오 수행 불가.해결:네트워크 차단(page.route) 및 헤더 조작을 시도했으나, 서버 측의 강력한 정책(IP 기반 302 Redirect) 확인.전략 수정: CI 환경에서는 접속이 원활한 '오늘의집' 프로젝트만 수행하고, '무신사'는 로컬 전용으로 분리하여 파이프라인의 신뢰성(Stability) 확보.4. 🖥️ CI 환경 최적화 (Headless & Viewport)문제: CI 서버(Linux)는 UI가 없는 환경이라 렌더링 에러 발생 및 반응형 UI(모바일 뷰) 인식 오류.해결:headless: !!process.env.CI 설정을 통해 로컬/CI 환경 자동 감지.뷰포트 차이로 인한 요소 식별 실패를 방지하기 위해 반응형 UI에 의존적인 단계를 개선.🛠️ Tech StackCategoryTechnologyLanguageTypeScriptFrameworkPlaywrightCI/CDGitHub ActionsSecurityOpenSSL (Cookie Encryption)TargetOhou (Today's House), Musinsa
